LTE eNodeB Management Help

Main Page

‣ On the main screen, you can check the status of the entire base station.
‣ Please login to use all features.
☺ To login, enter your user and password and click the Login button.

☺ When you complete the login, it will be displayed as below and you can move to other pages freely by left and right keys.

‣ The overall status of the base station can be checked by the colored icon on the status bar.
   ★ Radio State
     Grean On: Radio output is On.
     Green Blinking: Initialization in progress to emit radio power.
     Green Off: Radio output is Off.
   ★ Service State
     Green On: LTE service is On.
     Green Off: LTE service is Off.
   ★ Alarm State
     Green On: No alarm raised.
     Red On: Alarm raised.
   ★ Sync State
     Green On: GPS Lock.
     Green Blinking: GPS Tracking.
     Green Off: GPS Unlock.
     GPS status can be checked in detail in the GPS status bar.

   ★ Link State
     Green On: Backhaul Link Staus, (IOPS) Connected Pulic Network.
     Red On: (IOPS) Conneted Local Network
‣ At the bottom of the page, checkbox for additional information.

   ★ The slide for eNodeB Kitting can be moved by checking the "kitting Enable(last slide)" checkbox in the main window..
   ★ When alarms occurred you can check the details of the alarm by checking the "Alarm Bar Enable" checkbox..

eNodeB Basic Inforamtion

‣ In eNodeB Basic Information, you can check or change the basic settings of eNodeB.
‣ You can check eNodeB's IP address, operating frequency, ECI(CellIdentity), PCI(PhyCellID),
    and you can check the neighbor cell information by clicking the neighbor cell button.
    A new window will pop up with the neighbor cell information as shown below.

‣ Also, if eNodB cannot be connected to the EMS, it can be used the internal parameter of eNodeB.
   ☺ You can change the frequency, ECP, or PCI, etc.
   ☺ To apply, please click "Save"→"Apply(Restart)" button sequentially.
‣ If you want to restart eNodeB without any changing the settings, click the "System Restart" button..

eNodeB Alarm Information

‣ In eNodeB Aalrm Information, you can check the current alarm of eNodeB and also chek the alarm history.
‣ To refresh the alarm information, pls click "Alarm Refresh" button.
‣ Also you can save the current alarm information and the alarm history.
   ☺ To save the current alarm information, pls click "Save Current Alarm" button.
   ☺ To save the alarm history, pls click "Save Alarm History" button.
   ☺ The files are will be saved as csv file.

eNodeB Radio Status

‣ In eNodeB Radio Status, you can check the radio transmission status of eNodeB. You can stop or start the radio transmisstion and change the attenuateion to decrease the output power.
‣ To refresh the radio status, pls click "Status Refresh" button.
   ☺ The information displayed in "Status" is the current radio output status..
   ☺ The radio output status will be displayed "Reset", "Initialized", "ON", or "OFF"
   ☺ It is a status that radio transmission is available when radio output status is displayed as "ON".
‣ To change the cell status, enter the values in the table below and click the Set button.

   ☺ The "Set" button is activated by inputting the changed value for each item..
   ☺ To turn on/off radio transmission, enter 1(On)) or 0(off) as the “Enable” item.
   ☺ To adjust the radio power level, enter the attenuation value for each antenna.

eNodeB Kitting Parameter

‣ The kitting parameter window is for setting up the information necessary for installation when installing eNodeB.
‣ The kitting parameter window can be moved by checking the "Kitting Enable(last slide)" checkbox in the main window.

‣ The main parameters to be set when installing eNodeB are as follows:.
   ★ EMS URL: URL of EMS to be registerd eNodeB
     IPv4 case: http://192.168.61.8:48708
     IPv6 case: http://[2001:db8:1000:baba:ae16:2dff:fe8c:179d]:48708
   ★ IPAddress: eNodeB IP address
     IPv4 case: 192.168.61.141
     IPv6 case: 2001:db8:1000:baba:cc4:7e00:100:742 혹은 2001:db8:1000:baba:cc4:7e00:100:742/120
   ★ Default Gateway: eBodeB Gateway Address
     IPv4 case: 192.168.61.1
     IPv6 case: 2001:db8:1000:baba::1
   ★ Subnt Mask(in case of IPv4): eNodeB IP Subnet Maske(eg: 255.255.255.0)
‣ Other parameters can be set according to the installation network environment.
   ★ On VLAN network environment, configure VLAN parameters.
   ★ On IPsec network environment, configure IPsec parameters.
   ★ If needed IOPS operation, Rectifier management, or RET mamagement, It should be cofigured the local network parameters. Normally you can use the parameter set as the default.
‣ To apply the configured the parameters, please click "Save"→"Apply"→"Restart" button sequentially.
